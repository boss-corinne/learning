Conditional Commands - Questions

These exercises are designed to expand on the topics introduced in the Conditional Commands chapter of the Shell Scripting training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

The purpose of these exercises is develop shell scripts that make decisions and execute different commands based on those decisions.

  1. Shell

    Write a script that will indicate if the login shell is the Korn Shell (check the value of the variable $SHELL). Make sure your scrpt works whether the Korn shell is installed as /bin/ksh, /usr/bin/ksh or some other location.

    Extend your script to say if the login shell is the Korn shell, Bash or another shell; for the latter print out the script name that is being used as the shell.

    Solutions

  2. At Home

    Write a script that will indicate if the current directory is the user's home directory. Do this initially by comparing variable values, and then write a second version that uses the pwd command to obtain the current working directory.

    Solutions

  3. Greetings

    1. Write a script that prompts the user to enter their name. Exit silently (with an error status) if the read fails; this will happen if the user types ^D rather than supply their name. If the user supplies a name display a simple greeting like Hello name otherwise display an error message saying that no name was supplied. Make sure your error is written to standard error and exit with an error status.
    2. Write a second script that will run the script you developed in the previous question and have it display an error message to standard error if the script fails.

    Solutions

  4. File Check

    Write a script that will use the first command line argument as a pathname and display a message to indicate whether this files exists, and if so if its a plain file, a directory of another type of file. Ignore any additional command line arguments and silently exit with an error status if no argument is supplied. Do this in two ways:

    1. by using multiple if commands
    2. by combining commands with && and not using an if command

    Solutions

  5. Number Range

    Write a script that will prompt the user to enter a number and then validate that the number is between 1 and 10. Display the number if it is within the valid range otherwise write an error message.

    Solutions

  6. Safe Search

    Write a script that wraps around grep to provide a simpler interface to using the command. Your script should:

    • accept a command line of the format search pattern files...
    • run a letter case insensitive search
    • support patterns starting with - by supplying the -- option to grep
    • ensure at least one filename is provided

    Solutions
