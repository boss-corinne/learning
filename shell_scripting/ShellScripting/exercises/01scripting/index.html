Introduction to Shell Scripts - Questions

These exercises are designed to expand on the topics introduced in the Introduction to Shell Scripts chapter of the Shell Scripting training course.

You can attempt the exercises in any order but the later exercises tend to be more complex and require more time to complete. You are not expected to complete all the exercises in the time available. Just do as many as you can do, or want to do, and try to find time after the course has finished to return to complete these exercises. If you are unsure what to do, or need advice as you work through the exercises then please ask your instructor for additional information.

The purpose of these exercises is to create and execute simple shell scripts. You will find it beneficial to create a working directory for this course and place your scripts in that directory.

  1. Login to Linux

    Your instructor will provide you with login details for your environment. You will need to know:

    • your username
    • your password
    • the hostname if you are using SSH

    Using this information login using one of the following alternatives dependent upon your training environment:

    • If you have a Linux Desktop environment login through the desktop and launch a Terminal window or xterm window as demonstrated by the instructor.

    • If you are using a Microsoft Windows PC then start an SSH client as demonstrated by the instructor and login to the Linux server.

    From the shell prompt how do you identify your default shell - are you using Bash or one of the Korn Shells?

  2. Writing a Shell Script

    Perform the following actions which will help you understand how to write and run shell scripts:

    1. On a single command line run multiple commands to create a working directory for this exercise and then change to this new working directory.
    2. In this working directory write a shell script that will display the path of the current directory and then a long listing of the current directory - make sure you don't give the script the same name as a standard command (something like pls would work).
    3. Execute your shell script by passing it as a parameter to the default shell (/bin/sh).
    4. Make your shell script executable and run it using a relative path name (don't rely on the current directory being on the search path).
    5. Modify your search path to append the working directory containing your script and then run your script just by typing its name.
    6. Add a shebang comment to the start of the script to specify which shell you want to use: choose the correct absolute pathname for either ksh or bash according to your shell preference. You will need to use the type or whence command to find the location of the executable for your shebang line. Run your script - you may want to add a command to the script to display which shell is used to run the script.
    7. Update your script to pipe the ls output through either more or less, check your script works with a large directory such as /usr/bin.

  3. Shell Debugging

    Create a shell script called features with the following lines: 

    date
echo Current directory: $PWD
ls -l *
ls -l ~ | grep '^d' | more

    Using this script work through the following steps:

    1. Run your preferred shell and pass this script as a command line argument.
    2. Run the same command line as before but supply the shell's -n option and explain why there is no output.
    3. Run the original command but this time use the -v option and explain the output.
    4. Finally run the original command with the -x option and explain the output.

  4. Reverse Words

    Write a script that prompts the user to enter a line of data, read the words on this line into three variables from the user and then display the words in reverse order. Don't worry if the user enters more or less than three words, simply display the three variables in reverse order.

  5. Shell Environment

    Create the following script called setwh that will change create a new environment variable called WORK_HOME and the display the value of this variable both before and after setting its value: 

    echo "work home = $WORK_HOME"
export WORK_HOME=~/work
env | grep WORK_HOME

    Now work through the following steps and explain how the different ways of executing a shell script affect the environment used to define the WORK_HOME variable:

    1. Make sure that the WORK_HOME variable is not set in the current shell (use unset to do this).

    2. Use your preferred shell to run this script and supply a new directory. For example: 

      ksh setwh

      Verify that it displays out the value of WORK_HOME correctly. Now display the value of this variable using 

      echo $WORK_HOME

      it won't be defined, why is this?

    3. How would you run the setwh script so that it defines WORK_HOME in the current shell?
